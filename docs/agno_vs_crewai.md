# Agno vs CrewAI — Which One for DashForge AI?

## Side-by-Side Comparison

| Criteria | 🟣 Agno | 🔵 CrewAI |
|----------|---------|-----------|
| **Origin** | Rebranded from Phidata (2024) | Founded 2023, grew fast |
| **GitHub Stars** | ~20k+ | ~25k+ |
| **Philosophy** | Performance-first, code-first | Role-based teams, structured workflows |
| **Agent Instantiation** | ~2μs (ultra-fast) | Standard (slower) |
| **Multi-Agent** | Teams with delegation | Crews with role-based agents |
| **Workflow Style** | Flexible, code-driven pipelines | Sequential / Hierarchical processes |
| **Memory** | Built-in session + long-term + RAG | Short-term, long-term, entity memory |
| **Tool Ecosystem** | Growing, extensible | 40+ pre-built tools |
| **Multimodal** | Native (text, image, audio, video) | Text-focused |
| **LLM Support** | OpenAI, Anthropic, Google, Ollama | OpenAI, Anthropic, Google, Ollama |
| **Learning Curve** | ⭐⭐ Easier (minimal boilerplate) | ⭐⭐⭐ Moderate (more concepts) |
| **Docs Quality** | Good, improving | Very good, lots of tutorials |
| **Community** | Growing fast | Larger, more established |
| **Production Ready** | Yes (AgentOS, async, monitoring) | Yes (CrewAI AMP enterprise) |

---

## Decision: **🟣 Agno**

For DashForge AI specifically, Agno was chosen because:

1. **Less boilerplate** — Cleaner code for portfolio showcase
2. **Performance** — Ultra-fast agent instantiation helps hit < 60s target
3. **Simpler mental model** — Agent + Team + Tools vs Crew + Agent + Task + Process + backstory
4. **Multimodal future** — Native image/audio support for chat previews
5. **Portfolio differentiation** — Stands out vs the many CrewAI projects
